# Highlight Engine - 架构设计文档

## 1. 项目概述

### 1.1 项目简介
Highlight Engine 是一个用 Rust 重写的源代码语法高亮引擎，最初设计用于代码审查工具。该项目是从原始 C# 实现移植而来，提供了现代化的架构和更好的性能。

### 1.2 核心功能
- 解析源代码并根据语言配置文件应用语法高亮
- 支持多种编程语言通过 TOML 格式的配置文件
- 生成 HTML 输出和终端（ANSI）输出
- 提供基于 WPF 的演示应用程序（原 C# 版本）
- 命令行工具用于文件高亮处理

## 3. 核心模块设计

### 3.1 错误处理模块 (error.rs)

**设计目标**: 提供统一的错误类型和处理机制

**核心类型**:
```rust
pub type Result<T> = std::result::Result<T, HighlightError>;

pub enum HighlightError {
    Io(io::Error),
    Toml(toml::de::Error),
    Regex(regex::Error),
    InvalidProfile(String),
    LanguageNotFound(String),
    UnsupportedExtension(String),
}
```

**设计决策**:
- 使用 `thiserror` 库简化错误类型定义
- 提供详细的错误信息和上下文
- 支持从标准库错误类型的自动转换

### 3.2 语言配置模块 (profile.rs)

**设计目标**: 管理语言语法定义和配置加载

**核心数据结构**:
```rust
pub struct LanguageProfile {
    pub name: String,
    pub extensions: Vec<String>,
    pub delimiters: Vec<char>,
    pub back_delimiters: Vec<char>,
    pub ignore_case: bool,
    pub keywords: Vec<KeywordCollection>,
    pub single_line_blocks: Vec<CodeBlock>,
    pub multi_line_blocks: Vec<CodeBlock>,
    pub tokens: Vec<TokenDefinition>,
}

pub struct HighlightProfile {
    pub language: LanguageProfile,
    pub compiled_tokens: Vec<CompiledToken>,
}
```

**设计决策**:
- 使用 TOML 格式替代原始的 XML，提供更好的可读性
- 支持正则表达式模式匹配
- 预编译正则表达式以提高性能
- 支持转义序列和特殊字符处理

### 3.3 解析引擎模块 (parser.rs)

**设计目标**: 实现高效的语法解析和高亮标记

**核心算法**:
1. **行级解析**: 逐行处理源代码
2. **状态跟踪**: 维护多行块的状态（如多行注释）
3. **标记识别**: 识别关键字、字符串、注释等语法元素
4. **性能优化**: 使用快速查找表和字符串驻留

**关键数据结构**:
```rust
pub struct TextSegment {
    pub start_index: usize,
    pub length: usize,
    pub token_type: Option<Rc<String>>,
}

pub struct TextLineInfo {
    pub text_line: String,
    pub line_number: usize,
    pub segments: Vec<TextSegment>,
}
```

**性能优化策略**:
- ASCII 字符快速查找表
- 字符串驻留减少内存分配
- 预排序关键字用于二分查找
- 增量式多行状态处理

### 3.4 渲染器模块 (renderer.rs)

**设计目标**: 支持多种输出格式的渲染

**支持的渲染器**:
1. **HtmlRenderer**: 生成 HTML 输出
2. **TerminalRenderer**: 生成终端 ANSI 彩色输出

**设计特点**:
- 可配置的 CSS 类前缀
- 支持自定义主题
- HTML 转义处理
- 颜色格式转换（命名颜色 → 十六进制）

### 3.5 主题管理模块 (theme.rs)

**设计目标**: 管理颜色主题和样式配置

**核心功能**:
- 从 TOML 文件加载主题定义
- 支持命名颜色和十六进制颜色
- 提供默认主题（light, dark, monokai, solarized）
- 颜色查找和回退机制

### 3.6 字符串驻留模块 (interner.rs)

**设计目标**: 优化内存使用，减少重复字符串分配

**实现机制**:
- 使用 `HashMap<String, Rc<String>>` 缓存已驻留字符串
- 通过引用计数共享字符串实例
- 特别适用于重复的 token 类型名称

## 4. 设计决策详解

### 4.1 配置格式选择：TOML vs XML

**决策**: 使用 TOML 替代原始的 XML 格式

**理由**:
1. **可读性**: TOML 语法更简洁，易于人工编辑
2. **Rust 生态**: `toml` crate 提供优秀的序列化/反序列化支持
3. **类型安全**: 与 Rust 的类型系统更好地集成
4. **维护性**: 减少配置文件的复杂性

**权衡**:
- 需要迁移现有的 XML 配置文件
- 学习成本相对较低

### 4.2 错误处理策略

**决策**: 使用 `thiserror` + `anyhow` 组合

**理由**:
1. **库内部**: 使用 `thiserror` 定义具体的错误类型
2. **CLI 接口**: 使用 `anyhow` 提供上下文丰富的错误信息
3. **类型安全**: 编译时错误检查
4. **调试友好**: 详细的错误链和上下文

### 4.3 内存管理策略

**决策**: 使用引用计数 (`Rc`) 和字符串驻留

**理由**:
1. **共享所有权**: Token 类型名称需要在多个地方共享
2. **内存优化**: 避免重复的字符串分配
3. **性能**: 减少内存分配和释放的开销
4. **简单性**: 相比生命周期参数，`Rc` 更简单易用

**权衡**:
- 运行时引用计数开销
- 无法跨线程使用（单线程设计）

### 4.4 解析算法设计

**决策**: 基于状态机的增量解析

**理由**:
1. **多行支持**: 正确处理跨行的语法结构（如多行注释）
2. **性能**: 避免重复解析已处理的内容
3. **可扩展性**: 易于添加新的语法规则
4. **内存效率**: 逐行处理，避免大文件内存峰值

### 4.5 主题系统设计

**决策**: 基于配置文件的主题系统

**理由**:
1. **灵活性**: 用户可以自定义主题
2. **可维护性**: 主题与代码分离
3. **扩展性**: 易于添加新的主题
4. **兼容性**: 支持常见的颜色命名约定

## 5. 性能优化策略

### 5.1 解析性能

1. **快速查找表**: ASCII 字符使用布尔数组快速判断
2. **预排序关键字**: 使用二分查找提高关键字匹配效率
3. **字符串驻留**: 减少重复字符串的内存分配
4. **增量解析**: 避免重复解析已处理的内容

### 5.2 内存使用

1. **引用计数**: 共享不可变数据，减少内存复制
2. **延迟编译**: 正则表达式在需要时编译
3. **流式处理**: 逐行处理大文件，避免内存峰值
4. **字符串驻留**: 减少重复字符串的存储开销

### 5.3 I/O 优化

1. **批量读取**: 一次性读取整个文件
2. **缓冲写入**: 使用缓冲区减少系统调用
3. **异步支持**: 为未来的异步扩展预留接口

## 6. 扩展性设计

### 6.1 新语言支持

**添加步骤**:
1. 在 `languages/` 目录创建新的 TOML 配置文件
2. 定义语言名称、文件扩展名、语法规则
3. 系统自动检测并加载新语言配置

**配置示例**:
```toml
name = "go"
extensions = [".go"]
delimiters = [" ", "\t", "(", ")", "{", "}", ";", ",", "."]
ignore_case = false

[[keywords]]
name = "keyword"
keywords = ["break", "case", "chan", "const", "continue", "default", "defer", "else", "fallthrough", "for", "func", "go", "goto", "if", "import", "interface", "map", "package", "range", "return", "select", "struct", "switch", "type", "var"]
```

### 6.2 新主题支持

**添加步骤**:
1. 在 `themes/` 目录创建新的 TOML 主题文件
2. 定义颜色方案和 token 颜色映射
3. 通过 CLI 参数或 API 使用新主题

### 6.3 新渲染器支持

**扩展方式**:
1. 实现 `Renderer` trait
2. 添加新的输出格式支持
3. 在 CLI 中注册新的渲染器

## 10. 总结

Highlight Engine 采用现代化的 Rust 架构，在保持与原始 C# 版本功能兼容性的同时，提供了更好的性能、安全性和可维护性。通过模块化设计、配置驱动的扩展机制和全面的错误处理，该系统具有良好的可扩展性和易用性。

项目的核心优势在于：
- **类型安全**: Rust 的类型系统在编译时捕获许多错误
- **内存安全**: 无需担心内存泄漏和悬空指针
- **高性能**: 零成本抽象和高效的解析算法
- **可扩展性**: 基于配置的灵活扩展机制
- **现代化**: 使用现代 Rust 生态工具和最佳实践
- **关注点分离**: 语言语法定义与颜色主题配置完全分离，支持灵活的主题切换
